### 🔍 Deep Packet Inspection Engine

**What it solves:** High-throughput inspection of live network traffic — protocol classification, TLS SNI extraction, and per-flow state management at wire speed.

**Engineering challenges:**
- Designing a producer-consumer pipeline that keeps packet capture decoupled from analysis
- Maintaining flow-consistent state under concurrent processing without lock contention
- Parsing variable-length protocol headers across TCP/IP, TLS, and application layers

**Stack:** `C++` `PCAP` `Multithreading` `TCP/IP` `TLS SNI` `Flow Tracking`

**Impact:** Demonstrates deep understanding of network internals and systems concurrency — relevant to DPI, observability, and network security tooling.

### ⚙️ ToyLang Compiler & Interpreter

**What it solves:** An end-to-end compiler pipeline for a custom language — from source text to AST to execution, surfaced via a web UI.

**Engineering challenges:**
- Designing a grammar and hand-rolling a recursive descent parser
- Building a correct AST traversal and evaluation engine
- Exposing execution state cleanly through a React + Flask interface

**Stack:** `Python` `Flask` `React` `Compiler Design` `Parsing` `AST`

**Impact:** Shows first-principles understanding of how languages work — directly applicable to developer tooling, interpreters, and language server work.

### 🔗 Collabify — Real-Time Collaboration Platform

**What it solves:** Multi-user, real-time document collaboration with low-latency synchronization — think Google Docs, built from scratch.

**Impact:** Demonstrates practical command of event-driven architecture and real-time systems — relevant to collaborative tools, multiplayer platforms, and live data products.

## Why Software Engineering?

The most interesting engineering problems sit at the intersection of scale, correctness, and speed. I'm drawn to:

| Domain | What interests me |
|---|---|
| **Distributed Systems** | Consensus, consistency, and the tradeoffs that make large systems reliable |
| **Computer Networks** | How packets actually move — protocols, congestion, routing, and inspection |
| **Backend Infrastructure** | The plumbing that makes products work at millions of requests per second |
| **Systems Programming** | Writing code that talks directly to the OS — memory, threads, I/O |
| **Real-Time Applications** | Low-latency synchronization, event-driven design, live data pipelines |
| **Developer Tools** | Compilers, debuggers, language servers — the software that builds software |
| **Scalable Architecture** | System design that holds up when load × 100 |

I'm looking for teams building hard things in these spaces — where strong CS fundamentals and a willingness to go deep actually matter.
